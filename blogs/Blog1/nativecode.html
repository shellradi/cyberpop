----COMPUTER ARCHITECTURES
#REGISTERS
- small areas of memory in the cpu itself where you can store and manipulate data


______----
#MEMORY
 - is where most of the state of an application will be stored.

*PHYSICAL MEMORY (actual ram you have available)

*VIRTUAL MEMORY (an abstraction that allows one to access memory).
  
 This is how normal applications access physical memory and its how processes are sepparated from each other  and how we share code between processes.

 - virtual memory contains page table which splits the address space of a process into a series of pages, which are either unmapped or points to some physical memory address.
 -

 - virtual memory contains page table which splits the address space of a process into a series of pages, which are either unmapped or points to some physical memory address.
 -Each page has permissions like read /write
-its how the cpu knows if it can  perform those operations against a given memory address.

#STACK
 -values local to a function are stored in a part of memory called stack, which grow down .

 -When you call a function you create a new stack frame. You store the previous stack pointer somewhere then subtrsct from it to give yourself space in which to store any data required for the current function.
 - once you are done running the function you can restore original stack pointer.


--------------------
#ENDIANESS
-------
The endianess of a cpu is determined by the order in which sepparate bytes of memory are stored  in memory.
This is needed because memory is accessed in 8 bits (bytes).


*LITTLE ENDIANESS - stores data with the least significant byte first in memory.

*BIG ENDIANESS - stores data with the most significant byte first.

#instructions
 - Basic unit of machine  code (performs single operation)  !add two registers together.
  !compare a register to a number.
  !Jump to another instruction.

#PROGRAM COUNTER- 
(instruction pointer) is a register that tells the cpu which instruction to execute next.
 - on most sytems you cant directly write to registers but rather use either a jump or call instruction to change it.

#CALLS
 - special type of jump which, before changing the program counter, first store the address of the next instruction either to the  stack or to a specific register. This is how you call a subroutine (function) .
 - When the subroutine is done, it uses a return instructions to set the program counter back to that stored address.


#SYSCALLS
 -How normal application tells the kernel to perform an operation e.g read/write

#STATIC LINKING- building  an application by including all code you use inside the main binary.

#SHARED LIBRARIES -building an application by calling out to other binaries. These are DLLs in windows , .so files on linux or dylibs on mac.
   -When executables are loaded  into the memory by the kernel it loads the referenced shared libraries into the address space as well.
  -Additionally, applications can load these libraries at runtime; this is often used to load plugins.

#Dissassemblers 
 -turn machine code into a readable text form.
  
*INTERACTIVE Dissamblers- typically gui-based and allow you to browse through the code and annonate it.
  !IDA pro
  !Hopper
  !Ghidra(open source)
*Non Interactive Dissamblers- typically give code listing of the code (dead listing).
  !Objdump
  !ODA
  !Radare2 

#DECOMPILERS
 Allows one to turn machine code to something resembling C. This often buggy and can hide important details, but makes reading the code mucg easier and faster.

#DEBUGGERS 
  -allows one to look at a running application and see what the state of its registers, memory etc at any point.
   !gdb (GNU Debugger)- for unix systems
   !WinDbg -Good for windows
   !most intersctive disassemblers also include debuggers.
